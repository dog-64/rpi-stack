# Repository Guidelines

## Роль: Senior DevOps Engineer

Работай с этим проектом как **опытный DevOps инженер** в совершенстве знающий:
- **Ansible** — best practices, роли, модули, идемпотентность
- **Raspberry Pi** — архитектура ARM, Raspberry Pi OS, специфика железа

## Project Structure & Module Organization

На данный момент репозиторий содержит только служебные файлы IDE в каталоге `.idea/` и не имеет исходного кода, тестов или ресурсов. Если вы добавляете код, закрепите структуру сразу и придерживайтесь ее. Рекомендуемый минимум:

- `src/` — основной исходный код проекта.
- `tests/` — модульные и интеграционные тесты.
- `docs/` — документация проекта и архитектурные заметки.
- `scripts/` — вспомогательные утилиты и автоматизация.

## Build, Test, and Development Commands

Команды сборки, запуска и тестирования в репозитории не обнаружены. Добавьте их и зафиксируйте в `README.md`. Примеры формата описания:

- `make build` — сборка проекта.
- `make test` — запуск полного набора тестов.
- `make lint` — проверка стиля и статический анализ.

## Coding Style & Naming Conventions

Единых правил форматирования в репозитории пока нет. При добавлении кода зафиксируйте:

- отступы (например, 2 или 4 пробела),
- стиль именования (например, `snake_case` для файлов, `PascalCase` для классов),
- используемые линтеры/форматтеры (например, `prettier`, `ruff`, `eslint`).

## Testing Guidelines

Тестовый фреймворк и требования к покрытию пока не определены. При добавлении тестов укажите:

- фреймворк (например, `pytest`, `jest`, `rspec`),
- соглашения по именам (`*_test.py`, `*.spec.ts`),
- минимальное покрытие (например, 80%) и команду запуска.

## Commit & Pull Request Guidelines

История Git отсутствует, поэтому стандартов сообщений коммитов и PR нет. Установите правила и зафиксируйте их, например:

- Сообщения коммитов в формате `type(scope): summary`.
- PR должен включать: краткое описание, ссылку на задачу, шаги проверки, скриншоты для UI-изменений.

## Security & Configuration Tips

Конфигурационные секреты не должны попадать в репозиторий. Используйте `.env` и добавляйте его в `.gitignore`.
