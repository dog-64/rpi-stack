# Проверка SSD перед миграцией

**Цель:** Проверить SSD, адаптер и кабель на работоспособность **ДО** начала миграции.

> **Почему это важно:**
> - Плохой адаптер/кабель → I/O ошибки → система не грузится
> - Медленный SSD → нет смысла мигрировать
> - Блок питания → SSD отключается при нагрузке
> - Bad blocks → данные портятся

---

## Краткая проверка (5 минут)

```bash
# 1. SSD обнаружен?
lsblk | grep sd

# 2. Скорость чтения (должно быть > 100 MB/sec)
sudo hdparm -t /dev/sda2

# 3. Ошибки I/O (должно быть пусто)
sudo dmesg -T | grep -E '(I/O|error|offline)' | tail -10
```

**Если всё OK → переходи к [миграции](ssd-migration-manual.md)**

---

## Подробная проверка

### 1. Проверка подключения SSD

```bash
# Показать все блочные устройства
lsblk -o NAME,SIZE,TYPE,LABEL,MOUNTPOINT

# Ожидаемый вывод:
# sda           120G disk
# ├─sda1        512M part bootfs
# └─sda2      119.5G part writable
```

**Если SSD не виден:**
- Проверь SATA кабель (подключён к адаптеру И SSD?)
- Проверь USB кабель (синий порт USB 3.0?)
- Перезапусти адаптер питания (отключи/включи)

---

### 2. Проверка скорости чтения

```bash
sudo hdparm -t /dev/sda2
```

| Результат | Значение |
|-----------|----------|
| > 150 MB/sec | ✅ Отлично (USB 3.0) |
| 80-150 MB/sec | ⚠️ Приемлемо (USB 3.0 медленный) |
| 30-80 MB/sec | ❌ Плохо (USB 2.0 или адаптер) |
| < 30 MB/sec | ❌ Очень плохо (проблема!) |

**Если < 100 MB/sec:**
- Адаптер работает в USB 2.0 режиме (попробуй другой порт/кабель)
- Слишком старый SSD (SATA II вместо SATA III)
- Проблема с питанием адаптера

---

### 3. Проверка ошибок I/O

```bash
# Ищем ошибки за последние 10 минут
sudo dmesg -T | grep -E '(I/O|error|offline|Device off)' | tail -20
```

**ОШИБКИ которые ищем:**
```
❌ I/O error, dev sda, sector XXXX
❌ Buffer I/O error on dev sda2
❌ Device offlined - not ready after error recovery
❌ sd 0:0:0:0: [sda] tag#X FAILED
```

**Если есть ошибки → проблема с адаптером/кабелем/питанием**

---

### 4. Проверка USB подключения

```bash
# Показать USB топологию
lsusb -t | grep -A2 "Mass Storage"
```

**Ожидаемый вывод:**
```
|__ Port 001: Dev 002, If 0, Class=Mass Storage, Driver=uas, 5000M
```

| Параметр | Значение |
|----------|----------|
| **5000M** | ✅ USB 3.0 SuperSpeed |
| **480M** | ❌ USB 2.0 (медленно!) |
| **Driver=uas** | ✅ Правильный драйвер |
| **Driver=usb-storage** | ⚠️ Fallback (медленнее) |

---

### 5. Проверка адаптера

```bash
# Показать USB устройства
lsusb | grep -i "storage\|sata\|jmicron\|asmedia"
```

**Известные хорошие адаптеры:**
| Модель | VID:PID | Статус |
|--------|---------|--------|
| JMicron JMS578 | 152d:0578 | ✅ Отлично (150+ MB/sec) |
| ULT-Best | 7825:a2a4 | ✅ Отлично |
| Ugreen JMS578 | (разные) | ✅ Хорошо (если питание OK) |

**Проблемные адаптеры:**
| Модель | VID:PID | Проблема |
|--------|---------|----------|
| VIA Labs VL817 | 2109:0715 | ❌ I/O ошибки |
| Cheap no-name | - | ❌ Нестабильно |

---

### 6. Проверка питания (ВАЖНО!)

**Симптомы нехватки питания:**
- SSD появляется/исчезает (`detected capacity change from X to 0`)
- Ошибки `Device offlined` при нагрузке
- Скорость падает после hdparm

**Решения:**
1. **Используй адаптер с внешним питанием** для SSD
2. **Хороший БП Raspberry Pi** — 5V/3A минимум
3. **Не используй USB hub** без питания для SSD

---

### 7. Проверка на bad blocks (опционально)

> **Занимает много времени!** Запускай только если SSD б/у или есть сомнения.

```bash
# Быстрая проверка (non-destructive)
sudo badblocks -sv -o badblocks.log /dev/sda2

# Полная проверка (долго!)
sudo badblocks -wsv -o badblocks.log /dev/sda2
```

**Результат:**
- `Pass completed, 0 bad blocks found` → ✅ SSD здоров
- `1 or more bad blocks found` → ❌ SSD умирает, замените!

---

## Быстрый чеклист перед миграцией

- [ ] SSD виден в `lsblk`
- [ ] Скорость > 100 MB/sec (`hdparm -t`)
- [ ] Нет I/O ошибок в `dmesg`
- [ ] USB 3.0 (5000M в `lsusb -t`)
- [ ] Драйвер uas (не usb-storage)
- [ ] Нет сообщений `Device offlined`
- [ ] Блок питания 5V/3A подключён

---

## Проблемы и решения

| Проблема | Решение |
|----------|----------|
| SSD не виден | Проверь SATA кабель, адаптер питания |
| Скорость < 30 MB/sec | Синий USB порт, другой кабель |
| I/O ошибки | Другой адаптер/кабель, проверь питание |
| Device offlined | Адаптер с внешним питанием |
| USB 2.0 (480M) | Синий порт USB 3.0 |
| Плохой адаптер | Замени на JMicron/ULT-Best |

---

## Проверка на разных хостах

Если SSD не работает на одном Pi — **попробуй на другом!**

```bash
# Проверить на sema (известно что работает)
ansible sema -m shell -a "sudo hdparm -t /dev/sda2"

# Если работает там → проблема с Pi или портом
# Если не работает → проблема с адаптером/SSD
```
