# Known Issues & Solutions Index

Этот документ содержит индекс всех известных проблем и их решений в нашем Raspberry Pi кластере.

## Категории проблем

### 🐛 Kubernetes / k3s

#### [k3s-node-time-sync-problem.md](k3s-node-time-sync-problem.md)
**Дата:** 2026-03-14
**Статус:** ✅ Решено
**Проблема:** Узел становится `NotReady` из-за неверного системного времени
**Причина:** Нет RTC в Raspberry Pi, NTP не работает → TLS сертификаты "not yet valid"
**Решение:** Автоматическая синхронизация времени в роли k3s перед установкой

---

### 💾 Storage & SSD Migration

#### [ssd-direct-setup.md](ssd-direct-setup.md)
**Дата:** 2026-03-15
**Статус:** 📝 Documentation
**Проблема:** Сложная миграция с SD карты на SSD с множеством потенциальных проблем
**Решение:** **Прямое развертывание** на SSD - прошить образ сразу на SSD, настроить EEPROM, применить network fix
**Преимущества:** Проще, надёжнее, чистая установка без артефактов миграции

#### [ssd-migration-issues.md](ssd-migration-issues.md)
**Дата:** 2026-03-07
**Статус:** ✅ Решено
**Проблема:** K3s-agent не может подключиться к серверу после миграции SSD
**Причина:** Конфликт `/etc/rancher/node/password` - новый пароль на узле, старый на сервере
**Решение:** Удалить узел с сервера, остановить агент, запустить заново (авторегистрация)

#### [ssd-filesystem-corruption.md](ssd-filesystem-corruption.md)
**Дата:** 2026-03-06
**Статус:** ✅ Решено
**Проблема:** Файловая система SSD повреждается при миграции
**Причина:** Некорректная загрузочная запись при использовании LABEL
**Решение:** Использовать PARTUUID вместо LABEL

## Рекомендации

### При добавлении нового хоста:

1. **Всегда** используйте Ubuntu 25.10
2. **Всегда** включайте NTP синхронизацию
3. **Перед миграцией на SSD** читайте все known issues
4. **Тестируйте** на одном узле перед массовым развертыванием

### Документация:

- Все проблемы должны быть документированы в этом индексе
- Каждая проблема должна иметь:
  - Дату обнаружения
  - Статус (открыта/закрыта)
  - Четкое описание и решение
  - Примеры команд
