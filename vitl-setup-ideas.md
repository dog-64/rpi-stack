# Идеи для настройки vitl (Raspberry Pi 4 4GB)

## Статус
- Ожидает корпус (заказан)
- Нет свободного порта на ethernet хабе
- Память: 4GB (меньше чем у остальных)

## Рекомендуемые роли для хоста с 4GB памяти

### 1. Pi-hole (DNS + Ad-blocking) ⭐ Рекомендуется
**Требования:** ~512MB RAM
**Плюсы:**
- Блокировка рекламы на уровне сети для всего кластера
- Легковесный
- Встроенный DHCP сервер (опционально)
- Web интерфейс для статистики

**Установка:**
```bash
curl -sSL https://install.pi-hole.net | bash
```

---

### 2. Мониторинг кластера (Prometheus + Node Exporter)
**Требования:** ~1GB RAM
**Плюсы:**
- Централизованный мониторинг всех хостов
- Метрики в реальном времени
- Можно добавить Grafana на другом хосте

**Роль:**
- Prometheus сервер на vitl
- Node exporters на всех других хостах

---

### 3. Backup сервер
**Требования:** ~256MB RAM (+ внешний диск)
**Плюсы:**
- Централизованное резервное копирование
- Rsync, Borg или Restic
- Легковесный процесс

**Задачи:**
- Ежедневный backup конфигураций с других хостов
- Backup важных данных
- Можно подключить внешний USB диск

---

### 4. NTP сервер для кластера
**Требования:** ~50MB RAM
**Плюсы:**
- Точное время для всех хостов
- Синхронизация времени внутри кластера
- Очень легковесный

**Установка:**
```bash
sudo apt install chrony
```

---

### 5. Syslog сервер (централизованные логи)
**Требования:** ~200MB RAM
**Плюсы:**
- Все логи кластера в одном месте
- Легко искать проблемы
- rsyslog или syslog-ng

---

### 6. WireGuard VPN Gateway
**Требования:** ~100MB RAM
**Плюсы:**
- Безопасный доступ к кластеру извне
- Легковесный современный VPN
- Минимальная нагрузка на CPU

---

### 7. Git сервер (Gitea)
**Требования:** ~512MB RAM
**Плюсы:**
- Легковесная альтернатива GitLab
- Web интерфейс
- Локальные репозитории для кластера

---

### 8. Home Assistant
**Требования:** ~1GB RAM
**Плюсы:**
- Контроллер умного дома
- Интеграция с Raspberry Pi Connect
- Автоматизация задач кластера

---

### 9. Комбинированная роль (Рекомендуется)
Можно объединить несколько легковесных сервисов:

1. **Pi-hole** (DNS + Ad-blocking)
2. **NTP сервер** (синхронизация времени)
3. **Syslog сервер** (централизованные логи)
4. **Backup скрипты** (с cron)

Все это займет ~1-1.5GB RAM, оставив запас для системы.

---

## Ansible Playbook для vitl

Создайте отдельный playbook для настройки vitl после подключения:

```yaml
---
- name: Настройка vitl как сервисного узла
  hosts: vitl
  become: yes

  roles:
    - pihole
    - ntp-server
    - syslog-server
    - backup-node
```

---

## Подключение к сети

### Вариант 1: Купить новый ethernet хаб
- Модель с 8+ портами
- Gigabit Ethernet
- Цена: ~$20-30

### Вариант 2: USB Ethernet адаптер
- USB 3.0 to Gigabit Ethernet
- Цена: ~$15-20
- Подключить к одному из Pi с двумя сетевыми интерфейсами

### Вариант 3: WiFi (временно)
- Использовать встроенный WiFi
- Не рекомендуется для продакшена
- Только для начальной настройки

---

## После получения корпуса

1. Установить в корпус
2. Подключить к сети (ethernet или WiFi)
3. Добавить IP адрес в inventory.yml
4. Перенести из группы `inactive` в `active`
5. Запустить playbook для первичной настройки
6. Настроить выбранные сервисы

---

## Примерная конфигурация в inventory.yml

```yaml
vitl:
  ansible_host: 10.0.1.XX  # Назначить после подключения
  hostname: pXX
  model: "Raspberry Pi 4"
  memory: "4GB"
  role: "service-node"
  services:
    - "Pi-hole"
    - "NTP Server"
    - "Syslog Server"
    - "Backup Node"
```
